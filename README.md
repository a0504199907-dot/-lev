# Business Table — לעסקים יש שולחן אחד 🟠

פלטפורמת ווב מלאה בעברית (RTL מקצה לקצה) לניהול **לוח העסקים הפיזי השבועי**:
אתר ציבורי עם לוח משבצות אינטראקטיבי והדמיית מודעה בלייב, טופס שריון מהיר,
ארכיון מהדורות — ודשבורד ניהול מלא לאישור בקשות, חסימת משבצות והעלאת הארכיון.

> האתר הקודם של "לב עיצובים" (Vite) שמור בענף `main`. הענף הזה מכיל את פלטפורמת Business Table.

## סטאק

- **Next.js 15** (App Router) + **React 19** + **TypeScript**
- **Tailwind CSS** — צבע מוביל כתום-קורל `#FF5F38`, אפור פחם `#1E293B`
- **lucide-react** — אייקונים; קומפוננטות UI בסגנון shadcn/ui (`components/ui`)
- **Supabase** — בסיס נתונים (Postgres + RLS) ו-Storage לקבצי מודעות וארכיון
- **Heebo Variable** — גופן עברי (@fontsource)

## הרצה

```bash
npm install
npm run dev      # http://localhost:3000
npm run build    # בניית פרודקשן
npm start        # הרצת הבנייה
```

ברירות המחדל בקוד כבר מצביעות על פרויקט ה-Supabase `lvds-production`
(הסכימה הוחלה שם כמיגרציה `business_table_platform`), כך שהאתר עובד מיד
לאחר clone. לדריסה — העתיקו את `.env.example` ל-`.env.local`.

| משתנה | תפקיד |
|--------|--------|
| `NEXT_PUBLIC_SUPABASE_URL` | כתובת פרויקט ה-Supabase |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | מפתח publishable (בטוח לצד לקוח) |
| `ADMIN_PASSWORD` | סיסמת הכניסה ל-`/admin` (ברירת מחדל: `business2026`) |
| `NEXT_PUBLIC_DEMO` | `1` = מצב דמו מקומי ללא Supabase |

## מה יש באתר

### האתר הציבורי (`/`)
- **Hero** — "החשיפה הפיזית הכי חזקה לעסק שלך באזור" + גלילה חלקה ללוח.
- **הלוח החי** — גריד רספונסיבי (8 עמודות בדסקטופ, 4 במובייל) המשקף את לוח
  המודעות הפיזי; משבצות בגדלים 2×2 / 2×1 / 1×1 עם מחיר לשבוע.
- **יומן זמינות** — בחירה בין 4 ימי הראשון הקרובים; משבצת עם שריון מאושר
  (או חסימה ידנית) מוצגת כ"סגור", כולל המודעה האמיתית של המפרסם.
- **הדמיה בלייב** — העלאת גרפיקה בטופס שותלת את התמונה מיידית בתוך המשבצת
  הנבחרת בגריד, לצד שאר המפרסמים.
- **טופס שריון מהיר** — שם העסק, בעל העסק, טלפון, קובץ המודעה; שמירה
  ב-DB (סטטוס `pending`) והעלאת הקובץ ל-Storage.
- **ארכיון מהדורות** — פתיחה/הורדה של קובצי PDF + **סליידר לוגואים** נע של
  לקוחות קבועים.

### דשבורד הניהול (`/admin`)
מוגן בסיסמה (עוגיית httpOnly דרך middleware).

- **בקרת לוח ויזואלית** — בחירת שבוע, מצב כל משבצת (פנוי / ממתין / תפוס),
  חסימה/שריון ידני ללקוחות קבועים ושחרור משבצות.
- **תור בקשות לאישור** — כל בקשה עם פרטי הלקוח ותצוגה מקדימה ענקית (כולל
  זום); "אשר שריון" נועל את המשבצת ודוחה אוטומטית בקשות מתחרות; "דחה" משחרר.
- **מעלה הארכיון** — העלאת PDF + תמונת תצוגה שמתפרסמים מיידית בארכיון הציבורי.

## מבנה

```
app/
  page.tsx            דף הנחיתה הציבורי
  admin/page.tsx      דשבורד הניהול
  admin-login/        מסך כניסת מנהל
  api/admin/          login / logout (עוגיית ניהול)
components/
  site/               Hero, BoardSection (הלוח החי+טופס), ארכיון, סליידר...
  admin/              בקרת לוח, תור בקשות, מנהל ארכיון
  board/              SlotCell + WeekPicker (משותפים לציבורי ולניהול)
  ui/                 קומפוננטות בסיס בסגנון shadcn/ui
lib/
  data.ts             שכבת הנתונים (Supabase + נפילה למצב דמו)
  supabase.ts, demo.ts, weeks.ts, types.ts, config.ts
supabase/schema.sql   הסכימה המלאה + RLS + Buckets + Seed
middleware.ts         הגנת סיסמה על /admin
```

## מצב דמו

בלי חיבור ל-Supabase (או עם `NEXT_PUBLIC_DEMO=1`) האתר עובר אוטומטית
לנתוני הדגמה שנשמרים ב-localStorage — כל הזרימות (שריון, אישור, ארכיון)
עובדות מקומית, כולל מודעות לדוגמה.

## הערת אבטחה

בהתאם למפרט אין מערכת משתמשים; מדיניות ה-RLS פתוחה כדי שהדשבורד יעבוד עם
מפתח ה-publishable, וההגנה על `/admin` היא בסיסמה בלבד. לקראת הרחבה מומלץ
לעבור ל-Supabase Auth ולצמצם את המדיניות (למשל: עדכון/מחיקה רק ל-role
מאומת). קובץ הסכימה מסמן את הנקודות הרלוונטיות.
